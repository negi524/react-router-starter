# モックアップ作成

会話に貼り付けられた画像1枚を元に、事業側との認識合わせ用の静的モック画面を React Router プロジェクトに1ページ生成する。全てダミーデータで、JS の機能は実装しない（HTML 標準タグの素の挙動は妨げない）。

このモックは使い捨てではなく、**後に本格的な Web ページへ育てる土台**になる前提。本番の React Router 構造（`web/app/routes/` 直下）に最初から馴染ませる。

## 前提・対象プロジェクト

- React Router プロジェクト（v8 + Tailwind CSS v4 + TypeScript、pnpm 管理）は `web/` サブディレクトリにある。
- ルートファイルは `web/app/routes/` に置き、`web/app/routes.ts` に登録する。
- コマンド（typecheck / lint）は `web/` ディレクトリで実行する。

## 入力

- 直前の会話に貼り付けられた画像1枚を、Read（視覚）で読み取る。MCP やセットアップは不要。

## ワークフロー

1. **画像を確認する。** 会話中に画像がなければ、貼り付けるようユーザーに促す。
2. **ファイル名を提案する。** 画像から画面の用途を推測し、ファイル名を提案する（質問ではなく提案）。例: 応募者一覧 → `applicants.tsx`。ユーザーが直したい場合のみ修正を受ける。
3. **ルートファイルを新規作成する。** `web/app/routes/<提案名>.tsx` を作成する。**既存ファイルは上書きしない。** 同名があれば別名を提案する。
4. **ルートを登録する。** `web/app/routes.ts` に `route("<path>", "./routes/<提案名>.tsx")` を追記する。`index` や `authLayout` は触らない。
5. **画像を再現する。** 読み取れる内容はそのまま再現する。**読み取れない箇所のみプレースホルダー表記**（`〇〇〇`、`XXX` 等）。リストの行数等が不明ならプレースホルダー行を数行置く。
6. **検証する。** `web/` で `pnpm typecheck` と `pnpm lint` を実行し、コードが壊れていないことを確認する。
7. **完了報告する。** 生成ファイルパス、追記したルート、ブラウザでの確認方法（`web/` で `pnpm dev` → `/<path>` を開く）を伝える。

## 「JSの機能を実装しない」の定義

- **実装しないもの（JS の機能）:** `onClick` 等のイベントハンドラ、`useState` 等の状態管理、データ操作・フィルタ・ソート、`Link` / `navigate` による画面遷移、API 連携。`<button>` も使わない（ボタン風の見た目は `<a href="#">` で作る）。
- **そのまま許容するもの（HTML標準タグの素のブラウザ挙動）:** `<input>` / `<textarea>` への入力、`<select>` の展開、checkbox / radio の選択、`<details>` / `<summary>` の開閉、`<a href>` のアンカー、スクロール、CSS `:hover` 等。これらは JS の独自実装ではないため対象外。`preventDefault` 等で標準挙動を止めることもしない。

## コーディング規約（最小ルール）

- 1画像 = 1ルートファイル。
- **コンポーネント分割しない。** サブコンポーネントや別ファイルへの切り出しはせず、1つの tsx ファイル内に全て直書きする（モックなので1枚で十分）。
- ダミーデータは同ファイル先頭に `const` で直書きする。
- 上記「JSの機能を実装しない」定義に従い、JS ロジックは書かない（標準タグ挙動は妨げない）。
- **ボタン風の見た目の要素は `<a href="#">` で作る。** `<button>` は使わず、アンカータグでボタン UI を表現する（クリックしても何も起きない素のアンカー挙動）。`href` は `"#"` を指定する。
- 素の Tailwind v4 + react-icons のみ使用する。**依存追加禁止**（shadcn 等を入れない）。
- 既存コードのスタイルに合わせる:
  - コンポーネントは `export default function <Name>()` で定義する。
  - `meta` 関数でタイトルを設定する（`web/app/routes/index.tsx` に倣う）。
  - className は Tailwind ユーティリティ中心。`dark:` 対応は任意。
  - アイコンは `import { FiXxx } from "react-icons/fi"` 形式で import する。

### ファイルの雛形

```tsx
import { FiSearch } from "react-icons/fi";

// ダミーデータは先頭に const で直書き
const applicants = [
  { id: 1, name: "山田 太郎", job: "フロントエンドエンジニア", status: "選考中" },
  { id: 2, name: "佐藤 花子", job: "〇〇〇", status: "書類選考" },
  // 行数が不明ならプレースホルダー行を数行置く
];

export function meta() {
  return [{ title: "応募者一覧" }];
}

export default function Applicants() {
  return (
    <main className="container mx-auto p-4 pt-16">
      {/* 画像を再現した静的マークアップ。onClick 等は付けない */}
      {/* ボタン風の要素は <a href="#"> で作る（<button> は使わない） */}
    </main>
  );
}
```

## スコープ外（やらないこと）

- 複数画面・画面間遷移の生成（1回につき1画面）。
- データ操作・フィルタ・状態管理・API 連携。
- UI ライブラリ導入、依存追加。
- 見た目のピクセル完全一致検証（一致確認はユーザーがブラウザで目視する）。
